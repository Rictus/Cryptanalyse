Command line reference

DeskJS has the ability to execute JavaScript code and files right from within the command prompt in Windows. In order to achieve this, you'll either need to specify the path to the current DeskJS's folder right from within the command prompt (cmd in short), or alternatively and much easier, is by just opening DeskJS's folder (where the application resides), and typing within the address-bar (the one at the top) cmd then press on ENTER. Also, make sure it's the only text in the address-bar. See the example below:

You can then type deskjs in the command prompt followed by a space, then either of the below options (or none at all), followed by a space, then your code (in single or double quotes):

  • -o launches the app then executes code
  • -s silently executes code without launching the app but displays error dialogs if at all any error is found in your JavaScript code
  • -sh silently executes code but hides error dialogs

As an addition, you can also check the version of the application through the command-line option: -version or -v

NOTE: By default, code is silently executed (won't launch the app) in the command-line, and if at all any errors result from the code or file typed, a dialog will always popup displaying the Error and Line-number information to help ease the burden of debugging your scripts. Also, you won't need to type exactly how the app's name is written, meaning that it's not case-sensitive, so a name like DESKJS, DeskJS, or even deskjs will still work.

Examples

If you want to use DeskJS by specifying the app's path from within the command line, type the path of the application followed by a space, then the app's name.

For example:

C:\Users\admin\Desktop\DeskJS deskjs "code here"

If you're specifying options you can do either of the below:

C:\Users\admin\Desktop\DeskJS deskjs -o "code here" C:\Users\admin\Desktop\DeskJS deskjs -s "code here"

You can also cd (change directory) to the app's directory/folder by typing cd "path\to\folder", then pressing on ENTER. For example:

cd C:\Users\admin\Desktop\DeskJS

...then type:

deskjs "your code here"

Options still do apply here:

deskjs -o "code here" or deskjs -s "code here"

If you want to view your outputs via the command-line, you'll essentially need to use the basic JavaScript popup-boxes like an alert box since no results are returned directly to the command prompt.

A good example is, let's say you want to calculate 250 x 70, all you'll need to write is:

deskjs alert(250*70)

...then the result will be displayed with the alert box.

This trick will let you devise better ways of getting your results back even when you're bored somewhere and want to do some quick scripting. You may however make your work easier by configuring other text editors to return your results back with ease. You will be guided on such an example later on in this guide.

Keep in mind that double quotes ("") should be used whenever you're writing code that includes spaces which are not within any quotes whereas single quotes should always be used whenever you're wrapping simple strings such as alert('hello world'). For example, say you want to write var x = 'john'; alert(x) in the command prompt. You'll need to wrap the code with double quotes like this:

deskjs "var x = 'john'; alert(x)"

Also, if you're wrapping your code within double-quotes, then make sure that they do not appear anywhere inside your code. For example:

deskjs "var x = "john"; alert(x)" - this will produce an Error

This is because the added quotes surrounding john will split your code apart and make the Interpreter select "john" as the code to interpret rather than the whole code written. In essence it will look like you're doing this:

deskjs "john"

Your code will therefore produce a ReferenceError since "john" is not defined or declared.

However, if you're including code which has no spaces, then any strings you write in the code should be wrapped within single quotes. A good example is this:

deskjs alert('hello world!')

So always try to follow these guidelines as you use the command-line in executing your scripts.

Executing files

It's always an easier option to execute code in files that you've saved especially within the Console Scripts folder. Now, let's say you have a script file called hello world.js in the "Console Scripts" folder. Assuming you're within the application's folder in the command prompt, you can easily execute it by typing:

deskjs hello world

Notice that you won't even need to type the extension .js as DeskJS will scan through your saved script files and check to see if the file you've specified is of JavaScript type. Yet still, you can specify the extension if you so wish to. This will also apply for script files stored elsewhere.

To execute script files located elsewhere, you'll only need to specify their paths, for example:

deskjs C:\Users\admin\Desktop\hello world

Also to note, you can even execute script files by dragging-and-dropping them into the application's icon (or shortcut if it's in your Desktop or elsewhere) if you simply want to run some code in a script file without directly launching the application. By default, if any errors do exist, you'll be alerted about them and where they occur in the file with regards to their line-numbers.

If you love geeking things out, you can even write batch scripts or configure other editors like Notepad++ to execute JavaScript code via DeskJS quickly and seamlessly. However, even for beginners, it's never impossible to explore beyond the boundaries no matter how difficult and daunting the task may be! Configuring Sublime Text with a DeskJS build system

What is Sublime Text? It's an amazingly powerful editor for code and prose writing. Plus, it's supported on all major Operating Systems namely, Windows, Mac, and Linux. You can download it via it's official homepage: www.sublimetext.com.

With the numerous number of developers getting a hand on Sublime Text's powerful features right at their fingertips, well, it was worth writing a mini guide on how to achieve running JavaScript code through DeskJS right from within Sublime Text.

First, you'll need to download a copy of Sublime Text (a portable or installer version). Then, launch the app, go to the Tools menu, then select Build System. At the very bottom of the menu that appears, you'll see New Build System.... Select it.

From there on, you can choose to replace the currently set text with this template format:

{

    "cmd": ["path to DeskJS.exe","options","$file"],

    "selector":"source.js"
}

Below is an explanation of the above format:

  • "path to DeskJS.exe" - This represents the place where you will insert the path to DeskJS. An example is "C:/Users/admin/Desktop/NewFolder/DeskJS.exe"
  • "options" - This represents the place where you will insert an option related to executing your JavaScript code as discussed at the beginning of this guide. To do this, you can include an option like "-s" or "-sh". This is however optional so you may not even need it if you want to see error messages by default. If however you want to hide the error messages, just include "-sh" and that will be done for you.
  • "$file" - This is the place where Sublime Text inserts the path of your current JavaScript file. You can leave this property since Sublime Text dynamically changes this setting to whatever file it is you're currently in. It's what is referred to as a place-holder.
  • "selector":"source.js" - This setting means that DeskJS will only be running ".js" files which are JavaScript files. You can however change it to run other file types like maybe ".txt" by setting it to source.txt. However, if you're not into such deep details, you can keep it simple and clear by setting it to run JavaScript files only as shown.

A good example with the above format is this:

{

    "cmd": ["C:/Users/admin/DeskJS/DeskJS.exe","-sh","$file"],

    "selector":"source.js"
}

Just to be precise, make sure you don't remove the { and } at the top and bottom since this is a file format known as JSON (JavaScript Object Notation).

After this, you can save the file as DeskJS (or any name you wish) in the default location where Sublime Text will present to you. Normally, it displays to you the "Packages\User" folder. You can then save it there.

Now, to run any JavaScript file in Sublime Text, open any JavaScript file in Sublime Text or just create a new one if you have none. Then you can type some quick code, for example:

print("hello javascript").

After that, go to the Tools menu and select Build System; there, you'll see the new build system you created by the name you gave it (in our example, it's DeskJS). You can now select it and wait for the running process to complete. Alternatively, you can press on the shortcut keys Ctrl + B in Sublime Text to do the same.

A console will then appear right below the editor as your file is being run. Thereafter, the result will automatically be printed out for you in the console with the time it took for the process to be completed! Now that's fun...

If this doesn't work out for you, please feel free to consult by going to Sublime Text's forum or asking over at Stackoverflow.com to be assisted.
