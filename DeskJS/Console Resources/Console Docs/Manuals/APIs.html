DeskJS APIs

What's an API?

An API or Application Programming Interface is (in a very simple and straightforward explanation) a set of implemented methods that enable you to easily program a particular software or use some of its features developed for you as a programmer.

Below are some of the APIs implemented in DeskJS:

The notify() API

This API method allows you to print messages to the application's title area. It's a substitute for the print() method which simply prints messages to the console.

If you have tried changing any setting(s) in DeskJS, you may have noticed some messages being displayed right after the setting has been changed then after a while they disappear. Such messages are known as timeout messages and they are heavily used with settings. All you need to do is to type the API method notify() then your message goes in-between the two brackets.

Some few examples are:

notify("hello world!")

notify(12*3)

notify("i looove deskjs, it's really fun!")

The browse() API

This API method allows you to load any files or browse to any web page with DeskJS's inbuilt browser. It's also a very straightforward API so you won't need to include a rather huge line of parameters. This method takes only one parameter which is either a URL (Uniform Resource Locator) to a local file in your machine or a web address like www.google.com.

Below are a few examples using this API method:

browse("C:/Users/admin/Documents/hello world.html")

browse("www.google.com")

browse("http://www.w3schools.com/js/default.asp")

You can also view locally existing html files within your current script's directory/folder like the Console Scripts folder. This means that if you've saved your script file in the Desktop, you can load any existing html files that exist there. For example, to view an existing html file named "hello.html" that exists within your current script's directory, you can write this:

browse("hello")

Notice that you won't need to include the ".html" part of the file, but you can still add it if you want to. Html files existing in other sub-folders within your current script's directory can still be browsed-to simply by typing their relative paths based on your current script's directory. So, lets say the file hello.html now exists within the folder myfiles within your current script's directory; all you'll need to write is this:

browse("myfiles/hello")

This therefore means that you can nest html files within sub-folders that exist within your current script's directory.

This API method will always return True when called.

Though the browser is based on the native Internet Explorer from Microsoft (which means that some websites may fail to be rendered or loaded well in the browser), it can assist you in performing some tasks with much ease than rather than with a fully fledged browser application. Plus, you can do some actual web browsing with it if you so wish to.

The script API

Sometimes you may want to know or get the details of the script you're currently working on in order to handle some particular task. This is where the script API comes in handy.

The script API lets you get such information about your script(s) very easily. It is contextually-confined meaning that it's based on the current script you're working on. So whether you've saved your script in your PC or elsewhere, this API will always give you details concerning it.

Below is a list of properties and method available to use:

script.name - returns the script's name with its extension

script.nameOnly - returns the script's name without its extension

script.extension or ext - returns the script's extension (with a "." prefix - e.g .js)

script.path - returns the script's full path with its name and extension

script.folder or directory - returns the script's current folder/directory

script.include() - lets you include other JavaScript files in your current script (including their functions and other declarations such as variables)

We'll now discuss more on the script.include() API method.

One current limitation of JavaScript engines that exist outside the browser is the ability to include other script files in your current script. As usual, within the browser, you can do this simply by adding <script src="myscript.js"></script> within your html file with "myscript.js" being the script you're including then everything you declared within the script file will be included in your current script.

Now with DeskJS, you can include other JavaScript files anywhere in your code simply by adding script.include("file to include") within the current script file you're working on. You don't need to include the extension ".js" as this will be done for you. They are sort of treated like modules in DeskJS. A good example is if you want to include a script named hello.js that exists in your current script's directory. All you'll need to do is to write script.include("hello") in your current script. Plus, you won't need to save scripts so as to use this API method. This can be done even without having saved your code. Apart from including other scripts, you can also include JavaScript code. For example, you can include a function that you can then use in your code like this:

script.include("function add(x,y) { return x + y }"); add(2,3)

By default, the script.include() API lets you include script files that exist in your current script's directory, this meaning that its scope will be based on the current directory within which your script exists. So lets say you've saved your current script in the Desktop, where another script named "my other script.js" also exists. To include "my other script.js" in your current script, you can simply do this by writing:

script.include("my other script")

You can then reference the code defined in the script included. If let's say there's a function called add() that's declared in "my other script.js" such as the one shown there before, you can simply reference it as though it exists in the current script by typing: add(2,3)

Here's a sample usage scenario for this:

  • Script my other script.js content:

    function add(x,y) {
    return x + y;
}

  • Your current script's content:

    // include the script...
script.include("my other script");

// now call the function add() with values...
add(2,3);

It's that easy to implement and use.

But again at times, you may want to save your other scripts separately from your current script. Now let's shift the above scenario by saving the script "my other script.js" in a folder named "scripts". To include the script, all you'll need to do is write: script.include("scripts/my other script")

The / character lets you access other sub-folders within your current script's directory. This scenario can therefore be represented like this:

  • Script my other script.js content:

    function add(x,y) {
    return x + y;
}

  • Your current script's content:

    // include my script from the "scripts" sub-folder...
script.include("scripts/my other script");

// now call the function add()...
add(2,3);

This will be possible even for nested folders within your script's current directory.

Not only can you include one script file with the script.include() API method but can also include multiple script files with just one script.include() method. This is done by passing an array with the list of scripts you want to include. For example, if you have are two scripts in your current script's directory namely: "script1.js" and "script2.js", you can include them simply by typing:

script.include(["script1","script2"])

You may also represent the above scenario by creating an array and passing it as an argument to the API method:

var files = ["script1","script2"]

script.include(files)

Therefore, this API can take an array as an argument with the list of scripts you wish to include. It's still possible to include scripts contained within sub-folders from your current script like this:

script.include(["scripts/script1","scripts/script2"])

or even

script.include(["scripts/script1","scripts/script2", "others/script3"])

This API method can either return True if a script or scripts are successfully included or throw an IncludeError if there was an issue on inclusion.

It will therefore offer you a better way of including other scripts and even code to use in any of your saved scripts. You can even try including JavaScript libraries that have been built to extend JavaScript with additional tools and features.

The path API

As we've seen earlier with the script.include() API method, you're only limited to the current script's directory or folder when including other scripts. However, with this API, you can now go beyond the scope of any script's directory to even add scripts existing in other locations in your computer.

The path API gives you access to the all various paths in DeskJS (such as the snippets path) and your computer (such as My Documents folder) giving you a wider scope of inclusion for all your scripts. It's quite extensive but you won't need to memorize each and every pathname as it's supported in the completions list. For other editors such as Sublime Text, you can write a package supporting this API and others.

Below is a list of paths available with DeskJS:

path.app - returns the app's folder path

path.snippets - returns the app's snippets path

path.scripts - returns the app's scripts path (Console Scripts)

path.themes - returns the app's themes path

path.docs - returns the app's documentations path

path.settings - returns the app's settings file path

Below is a full list of system paths usable:

path.computer - returns the path to My Computer

path.desktop - returns the path to the Desktop folder

path.documents or path.personal - returns the path to My Documents folder

path.music - returns the path to My Music folder

path.videos - returns the path to My Videos folder

path.pictures - returns the path to My Pictures folder

path.recent - returns the path to the Recent folder

path.start - returns the path to the Start Menu folder

path.startup - returns the path to the Startup folder

path.system - returns the path to the System folder

path.systemx86 - returns the path to the System (x86) folder

path.programs - returns the path to the Programs folder in the Start Menu folder

path.programFiles - returns the path to the Program Files folder

path.programFilesx86 - returns the path to the Program Files (x86) folder

path.windows - returns the path to the Windows folder

path.sendTo - returns the path to the SendTo folder

path.appData - returns the path to the Application Data folder

path.adminTools - returns the path to the Administrative Tools folder

path.fonts - returns the path to the Fonts folder

path.cdBurning - returns the path to the file-CD writing area

path.netShortcuts - returns the path to the Network Shortcuts folder

path.netCache - returns the path to the temporary Internet files/cache folder

path.cookies - returns the path to the Internet cookies folder

path.resources - returns the path to the system's Resources folder

path.localResources - returns the path to the Local Resources folder

path.favorites or path.favourites - returns the path to the Favorites folder

path.history - returns the path to the Internet's History items folder

path.pubDocuments - returns the path to the Public Documents folder

path.pubDesktop - returns the path to the Public Desktop folder

path.pubStartup - returns the path to the Public Startup folder

path.pubStart - returns the path to the Public Start Menu folder

path.pubAdminTools - returns the path to the Public Administrative Tools folder

path.pubAppData - returns the path to the Public Application Data folder

path.pubPrograms - returns the path to the Public Programs folder in the Start Menu folder

path.pubProgramFiles - returns the path to the Public Program Files folder

path.pubProgramFilesx86 - returns the path to the Public Program Files (x86) folder

path.pubMusic - returns the path to the Public My Music folder

path.pubVideos - returns the path to the Public My Videos folder

path.pubPictures - returns the path to the Public My Pictures folder

All the main paths in the system are therefore exposed in DeskJS for you to use while scripting. Also, as you may have noticed, all the public folders begin with a prefix pub followed by the name of the public folder thereby making it easy for you to quickly type them.

We can now give an example of this API by combining it with the script.include() API method. Let's say you want to include a script file named hello.js located within your Desktop. This can easily be done by typing:

script.include(path.desktop + "hello")

If the script file is probably contained within a folder in your Desktop named myfolder, you can still include it by specifying the folder name followed by the / character then the script name:

script.include(path.desktop + "myfolder/hello")

Other examples are:

script.include(path.documents + "popups")

script.include(path.pubDocuments + "scripts/sample script")

script.include(path.appData + "samples/js/myfunctions")

If you happen to reference a path that doesn't exist in this API (like path.abcd), the Interpreter will return an undefined result meaning that the path you're trying to reference is not defined anywhere within the path API.
