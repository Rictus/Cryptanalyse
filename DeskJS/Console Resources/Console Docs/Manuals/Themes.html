Themes

What are themes?

Themes are small files that are able to define how DeskJS will look like based on their settings which are mostly color settings.

Themes are defined in JSON (JavaScript Object Notation) format, which is a very simplified file format as compared to the previously stated format known as XML. Theme files consists of different settings (for example: console_color) and their values (for example: "Blue") set besides them. Each setting uses this kind of format:

setting : "value"

This allows for a much simplified file definition based on how you want DeskJS to look like. A theme can also be put inside a folder together with its Icon package which will be discussed later in this guide. For beginners, well don't be afraid of this because even you can create your own theme(s) just by reading through this guide. Now lets go through the complete theme-file format with one of the default themes developed with DeskJS, which is the Light theme:

{

     "author" : "Willy Kimura",

     "icon_package" : "Default Icon Packages/Dark",

     "console_color" : "White",

     "outline_color" : "DarkGray",

     "app_title_color" : "Black",

     "title_labels_color" : "Black",

     "subtitle_labels_color" : "Black",

     "textbox_color" : "White",

     "textbox_fore_color" : "Black",

     "buttons_color" : "DarkGray",

     "buttons_fore_color" : "White",

     "editor_fore_color" : "Black",

     "editor_pointer_color" : "DimGray",

     "syntax_errors_color" : "Tomato",

     "runtime_errors_color" : "Green",

     "list_selection_color" : "DodgerBlue",

     "list_items_color" : "Black",

     "list_subitems_color" : "DimGray",

     "list_selected_items_color" : "White",

     "list_selected_subitems_color" : "White"

}

All themes should be saved with a .console-theme extension for them to be recognized in the Console Themes folder. The Console Themes folder is found within the Console Resources folder.

For the experienced JavaScript programmers, you may not even need to go through the whole Explanation section if you clearly understand what's going on and how the JSON file format works. The icon_package setting is however different from other theme settings as it defines the Icon package that will be applied based on your theme. You can read about it. Also to state, you can define colors based on their names, for example "Green", "LightSkyblue" or you may even use Hexadecimal or Hex color definitions such as "#00ff00" or "#994b00". The RGB color format is however not supported.

Explanation

We'll now go through the full list of settings as described above.

To begin with, if you wish to add some explanations for each or some of your settings, you can use comments which usually begin with double forward-slashes // just like in JavaScript. Also, notice that all the settings described above have a comma at the end except for the very last setting. This means that the last setting should not have a comma as it's to mean that that marks the end of all the settings in your theme file. Plus, you don't need to follow any order when defining your theme's settings as shown above. You may mix them together, but don't forget to include any setting when mixing them.

Let's begin:

Author

"author" : "Willy Kimura",

As stated before, this file format defines everything in it based on a setting : value style allowing each property to have its own name and value (also commonly know as name : value pairs). So for the above setting, you are allowed to set your name as the author of the theme. Notice that for Strings or statements, you will need to write them with double-quotes like: author : "James". This will apply for all the settings in all the themes you'll create.

Icon package

This is the setting that specifies the icon package you want to be used whenever a user applies your theme. By default, there are two icon packages that come pre-packaged with DeskJS, that is, the Dark and Light icon packages. The Dark icon package consists of dark (or black) icons that can be used with bright backgrounds while the Light icon package consists of light (or white) icons that can be used with dark backgrounds. You can find them in the Default Icon Packages folder stored within the Console Themes folder. If you just want to use any of the default icon packages created instead of creating your own icon package, you can specify by typing either of the below:

"icon_package" : "Default Icon Packages/Dark", "icon_package" : "Default Icon Packages/Light",

The forward slash indicates that you're accessing a certain folder in the Default Icon Packages folder. This is to mean that you can even nest other icon packages within the same folder.

Icon packages are simply folders which can consist of images of .png (Portable Network Graphics) or .ico (Icon) extension types. You can also mix different images of the mentioned extensions in one icon package. Your icon package will need to have all the relevant images as contained in either the Light or Dark icon packages. You can view the list of images there. Also, you will notice that there's a folder known as Dialogs in both the Dark and Light icon packages. This folder contains icons that the users can specify by default when using the alert, prompt, and confirm dialogs as we shall see later on in another guide. You may also wonder why other icons exist in that folder that are not quite that relevant to JavaScript's popup dialogs. Don't worry, they'll be of use in future... be assured of that!

If you want to package your theme in a folder together with your own icon package, make sure to reference your icon package inside your theme in this format:

"icon_package" : "Folder Name/Icon package",

For example, say you've created a theme named Hello.console-theme and inserted it into a folder you've called Hello Theme. Then, inside the folder, you've added the theme's icon package called MyIcons, you can specify this in your theme file's icon package setting as:

"icon_package" : "Hello Theme/MyIcons",

The / character specifies that you want to access a certain folder within the parent theme folder which in our example is Hello Theme. As an addition, you can also add a number of themes in your theme folder together with a number of icon packages since this model favours mixing different themes all in one theme folder.

Such an example can be put like this (based on our example):

Hello Themes/
            /MyDark icons
            /MyLight icons
            Hello Light.console-theme
            Hello Dark.console-theme

The above details the folder and file structure for including multiple theme-files and icon packages all in one folder. At this point, the Hello Themes folder can be referred to as a Theme package rather than a normal theme file.

Console Color

This basically specifies the background color of the console. The format is as below:

"console_color" : "White",

You can either use natural color-names such as White or Blue, or use Hex-color-names which are much more broad and specific such as #00ff00 which stands for color Green. To get started with learning how they work, just head over to w3schools Color Picker where you'll be able to mix different colors and see their results shown in this color format. You won't even need to learn them since all the heavy-lifting of knowing such colors will be done for you.

Outline color

This specifies the outline or border color of the console together will all the other windows in DeskJS. Below is the format:

"outline_color" : "DarkGray",

Application title color

This specifies the title color of DeskJS which is found in the Homescreen.

"app_title_color" : "Black",

Title labels color

This specifies the color of all the title labels in the application. A good example is the Tooltip which consists of a Title and a Description.

"title_labels_color" : "Black",

Subtitle labels color

This specifies the color of all the subtitle labels in the application. The same example of the Tooltip still applies here, letting you style the color of its subtitle.

"subtitle_labels_color" : "Black",

Textbox color

This specifies the background color of all the text-boxes (or input-boxes) in DeskJS.

"textbox_color" : "White",

Textbox fore color

This specifies the text color of all the text-boxes in DeskJS. This, in-short is to mean the color of the text you will type inside any textbox in the application.

"textbox_fore_color" : "Black",

Buttons color

This specifies the background color of all the buttons in DeskJS; from the alert popup-box to even the confirm popup-box.

"buttons_color" : "DarkGray",

Buttons fore color

This, just like the textbox, specifies the text found within the different buttons in the application. It sets the text color of all the buttons in DeskJS.

"buttons_fore_color" : "White",

Editor fore color

The editor refers to the place where you'll be writing your JavaScript code in the console. It's also referred to a the Console's editor. This setting specifies the color of the code you will type in the console's editor. Make sure that it's visible enough based on the background color you set for the Console.

"editor_fore_color" : "Black",

Editor pointer color

The editor pointer or console pointer character refers to the default two arrow-characters (>>) that point to the console's editor. You can change their color too by using the below setting with a color of your choice:

"editor_pointer_color" : "DimGray",

Syntax errors color

Syntax errors in JavaScript are those errors that result from not following the language or grammar rules correctly and as expected. A good example is, if you try to write an if conditional statement and you miss a closing curly brace } at the end, this will result to a syntax error since you haven't yet stated the end of the if statement. Below is its setting and a default value:

"syntax_errors_color" : "Tomato",

Runtime errors color

These errors are of different types or classes in JavaScript. However, they are different from Syntax Errors as they result from your code not having a meaningful fact (or logical/semantic rules) even though they follow the correct JavaScript grammar. A good example is when you are trying to call a function that does not exist or hasn't been created yet. This will result into a runtime error.

"runtime_errors_color" : "Green",

List selection color

This setting allows you to set the selection color that will appear on any item selected in a list. A good example of a list is the Results list. For the Light theme, once you start hovering over the results, you will notice a bluish color appearing. This is it's selection color which is set to DodgerBlue. Other types of lists are the themes and completions list.

"list_selection_color" : "DodgerBlue",

List items color

This will set the default color of the list items in any list. By default, the Light theme sets all list items with a Black color since the background color is White to make them easily visible.

"list_items_color" : "Black",

List subitems color

Though the Results list does not have sub-items, other lists such as the themes and the snippets list do have them. This setting will allow you to set any default color for their sub-items. By default, the Light theme comes with a less black color known as DimGray.

"list_subitems_color" : "DimGray",

List selected items color

As goes by its name, this setting allows you to set the color of any selected item in a list. If you set the selection color to be of a dark type (such as Black), then make sure the selected items color is of a rather light type (for example White).

"list_selected_items_color" : "White",

List selected subitems color

Likewise, for sub-items, ensure that the color follows the same rule as that of the items, though still you may play around with other colors to see how well they may turn-out to be. It's just a matter of mixing different colors and seeing their results in your theme.

"list_selected_subitems_color" : "White"

Previewing your themes

To be able to easily preview (or debug) your themes after you've ensured that your theme file has all the available settings in place, you can apply the theme from the themes list by pressing on Ctrl + L. Thereafter, to refresh your theme once you've made any changes to the color settings, just press on F6 from the console's editor which will quickly refresh your currently applied theme.
