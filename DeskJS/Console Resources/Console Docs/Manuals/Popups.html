JavaScript popup boxes

What are JavaScript popup boxes?

Popup boxes, or dialog boxes are specialized windows designed to get or output user-information to the screen. They are synchronous meaning that they need to be shown and executed before any other operation can be executed thereafter.

By default, there are three popup boxes designed in JavaScript that rest primarily in browsers. They are the alert, confirm, and the prompt dialogs. This is not to say that they need to exist in JavaScript for it to be complete. No, they are just minor additions to the language so that they may assist developers with getting user information and outputting information to users. By default, they exist in the DOM (Document Object Model) which is an addition to the language implemented in all modern browsers that allows web developers to program web-pages easily. You won't need to worry about this at all since the DOM is not a standard part of JavaScript. JavaScript is pure in, and of itself without the DOM. If you're a beginner, you can choose to learn the language itself without learning the DOM side of it. Just check out w3schools to learn more. Believe me, JavaScript will make you smile if you're willing to love and learn it. It's never impossible.

For most JavaScript consoles that exist outside the browser, they don't usually have these additions since they aren't a standard feature defined in the ECMA specification document. However, they are by default included in DeskJS so as to help users be able to do some Input/Output just like in the browser. By default, DeskJS's dialogs conform (or adapt) to the same format and return-types as those of the browser. They however have been extended to include more parameters as we shall see later in the section: Additional parameters. Below are some of the basic examples:

alert("hello world") : this returns an undefined value

confirm("are you sure?") : this returns true or false depending on what the user has clicked (either okay or cancel - still the Enter or Escape keys can be used)

prompt("enter your name") : this returns null if nothing has been entered or the user's input if the user typed-in something - the Enter or Escape keys can still be used here

Additional parameters

Alert

The alert popup has an additional three parameters, that is, title, icon and button text. They follow this format in declaration:

alert(message, title, icon, button_text)

The title parameter allows you to set a title for the alert popup box. The icon parameter allows you to set an icon for the alert box. It should be of String type. The button_text or simply button text lets you set the text for the alert's button.

By default, you can specify some pre-defined icons in the icon parameter. Below is a list of these icons:

  • "alert"
  • "action"
  • "information"
  • "input"
  • "key"
  • "message"
  • "okay" or "ok"
  • "question"
  • "quote"
  • "select"

If however you want to use an icon saved elsewhere in your computer, you can specify it's path followed by it's name and extension. An example is: C:/users/admin/Desktop/myicon.png

TIP: You can find these icons in the Default Icon packages folder, within the Dialogs folder of both the Dark and Light icons. Other themes can also have their own icons with the same names as the ones stated above.

These icons can also be used in the other JavaScript popup boxes. If however you want to use an external icon elsewhere in your computer, just type its path in the icon parameter.

Below is a list of examples with the stated parameters:

alert("hello world", "my title", "message") alert("hello world", "my title", "information", "click me!") alert("this is my icon", "hello icon", "C:/users/admin/Desktop/myicon.png") alert("you have completed the program!", "successfull", "okay", "close")

Confirm

The confirm dialog, just like the alert dialog also features two more parameters, that is the title and icon parameters, but with two additional parameters for setting the okay and cancel button texts. The same rules and icons also apply as in the alert dialog.

confirm(message, title, icon, button1_text, button2_text)

Below are some examples:

confirm("are you sure?", "please answer", "question", "yes") confirm("do you want to quit?", "quit execution...", "action", "yes", "no") confirm("do you want to begin?", "begin program", "alert", "sure", "later")

Prompt

The prompt dialog is probably one of the most used dialogs of them all. This is because it's the one mandated with getting a user's input. Just like the default parameters for the prompt dialog in browsers are designed, so is DeskJS's prompt dialog. It also has the most parameters than any of the above stated. It has additional two parameters that is the input_type and input_length which assist you in specifying the user's input better.

Below is the parameter-list format:

prompt(message, default_text, title, icon, button1_text, button2_text, input_type, input_length)

The default_text parameter refers to the default input that will be set and shown to the user right before the user begins typing his/her information.

The input type parameter allows you to specify whether you want the user to enter masked (password) text format, unmasked (normal) text format, numeric (number) text format, decimal number text format, uppercase text format, or lowercase text format. If you just want to let the user type anything, you can just leave this parameter blank (like "").

Below is the list of default values you can use for this parameter as stated:

  • "masked" or "password" or "m" : for passwords or masked user-inputs
  • "unmasked" or "normal" or "" : for normal user-inputs (not a must)
  • "numeric" or "number" or "num" or "n" : for number user-inputs (e.g. 123)
  • "decimal" or "dec" or "d" : for decimal number user-inputs (e.g. 12.567)
  • "upper" or "u" : for specifying upper-case user-inputs
  • "lower" or "l" : for specifying lower-case user-inputs

Some good examples with the said parameters are:

prompt("enter your name", "", "", "", "", "", "masked") prompt("enter your name in upper-case", "", "", "", "", "", "upper") prompt("enter your name in lower-case", "", "", "", "", "", "lower") prompt("type any number you want", "", "", "", "", "", "numeric") prompt("type any decimal number", "", "", "", "", "", "decimal")

Notice that you won't exactly need to specify other parameters if you don't want to. This especially goes for the two buttons' texts. If left empty, they'll default to their own pre-defined texts that is, okay and cancel.

The input length parameter allows you to specify the length of a user's input while they're typing. This parameter requires an Integer value type such as 10 or 50. The maximum input length is 2,147,483,647 so you don't have to feel constrained if you want the user to enter large strings. You may however experience some performance delays when typing much larger strings exceeding about 2.0 MB of size. Once you specify the length of input, a user will not be able to go beyond the set value thus limiting him/her from progressing beyond the set length.

Below are some examples illustrating the same:

prompt("type any decimal number below", "", "", "", "", "", "decimal") prompt("what is your favorite number?", "", "", "", "", "", "numeric") prompt("what was your score last year?", "", "", "", "", "", "decimal", 12)

With DeskJS therefore, the capabilities of the basic JavaScript popup-boxes have been extended to match the users' needs within the desktop arena. However, such additions to the popup boxes are only limited to DeskJS. They won't be recognized in the browser at all, unless otherwise implemented.
